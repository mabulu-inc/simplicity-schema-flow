## Description

Changes to `checks` expressions in YAML are silently ignored by the planner, drift checker, and executor. Check constraints are created on initial `CREATE TABLE` but never compared against the live database on subsequent runs. This breaks the declarative model — any modification to check constraint expressions after initial table creation requires manual SQL.

Three things must be fixed:

1. **Introspection**: Query live check constraints from `pg_constraint` (using `pg_get_constraintdef`) so the planner and drift checker have the current DB state.
2. **Planner**: When a check constraint's expression differs between YAML and DB, emit a `drop_check_constraint` + `add_check_constraint` operation pair (PostgreSQL does not support `ALTER CONSTRAINT` for check expressions).
3. **Drift detection**: Report check constraints whose expressions differ between YAML and DB as drift items.

After modifying a check constraint expression in YAML and running `schema-flow run`, the updated constraint must be applied. `schema-flow drift` must then report zero drift.

Close GitHub issue #12 after task completion.
