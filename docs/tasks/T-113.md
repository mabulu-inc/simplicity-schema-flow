## Description

After running `schema-flow run` on a fresh (empty) database, `schema-flow drift` reports all seed tables as having different seed data, even though the seeds were just inserted by `run`. The actual data in the database is correct — `SELECT * FROM table` returns the expected rows.

The root cause is likely a type coercion mismatch between YAML values and PostgreSQL query results. For example, YAML integer `1` vs PostgreSQL string `'1'` for serial columns, or similar numeric/string mismatches. The `diffSeeds` logic in `src/drift/index.ts` queries by PK and compares values, but the comparison may not account for type differences between YAML-parsed values and PostgreSQL result types.

The fix should normalize values to a common type before comparison (e.g., coerce both sides to strings, or parse PostgreSQL results to match YAML types based on the column's declared type).

After `schema-flow run` on a fresh database, `schema-flow drift` must report zero seed drift for all tables with seeds.

Close GitHub issue #10 after task completion.
