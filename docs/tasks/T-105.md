## Description

`schema-flow plan` always drops and recreates every RLS policy because the introspection misparses the `polroles` column from `pg_policy`. PostgreSQL stores policy roles as an OID array. The text representation is `{0}` (where OID 0 = PUBLIC). The introspection code treats the array literal as a string and truncates it to `{`, producing a mismatch that triggers phantom drop + create on every plan.

## Completion Notes

Fixed `getPolicies` SQL query in `src/introspect/index.ts` to properly resolve `polroles` OID array to role names. The previous approach used `LEFT JOIN pg_roles` with `array_agg`, which returned `{NULL}` for OID 0 (PUBLIC) — the pg driver parsed this as a raw string, yielding `"{"` instead of `"public"`. Replaced with a correlated subquery using `unnest(pol.polroles)` + `string_agg` with explicit `CASE WHEN oid = 0 THEN 'public'` mapping. Added 3 new tests and 1 assertion to existing test (total: 4 new assertions). All 1060 tests pass.
