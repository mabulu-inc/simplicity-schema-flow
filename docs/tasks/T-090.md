## Description

The planner emits `CREATE MATERIALIZED VIEW "<schema>"."<name>" AS <query>` without any idempotency guard. If the materialized view already exists (e.g., from a previous partial run or introspection mismatch), the statement fails with `42P07` (duplicate_table — PostgreSQL uses the same error code for mat views).

## Current state

- `src/planner/index.ts` lines 1555, 1572: `CREATE MATERIALIZED VIEW "${pgSchema}"."${mv.name}" AS ${mv.query}`
- PostgreSQL supports `CREATE MATERIALIZED VIEW IF NOT EXISTS` (PG 9.3+)

## Fix

Add `IF NOT EXISTS` to the `CREATE MATERIALIZED VIEW` statement:

```sql
CREATE MATERIALIZED VIEW IF NOT EXISTS "<schema>"."<name>" AS <query>
```
