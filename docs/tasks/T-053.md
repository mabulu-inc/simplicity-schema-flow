## Description

Add support for custom constraint names (PRD §4.1): (1) `primary_key_name` on TableSchema for custom PK constraint names (`CONSTRAINT pk_name PRIMARY KEY (...)`), (2) `unique_name` on ColumnDef for custom unique constraint names on single columns, (3) `name` field on ForeignKeyRef for custom FK constraint names. Parser must accept these fields. Planner must generate SQL with `CONSTRAINT name` clauses. Introspection should read constraint names. Drift should compare constraint names.
