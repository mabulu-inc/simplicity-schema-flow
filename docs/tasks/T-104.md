## Description

The introspection query does not extract `atttypmod` from `pg_attribute` for numeric columns. PostgreSQL stores precision/scale in `atttypmod` as `(precision << 16) + scale + 4`. Without reading this, schema-flow always sees `numeric` (no precision) and diffs it against the YAML's `numeric(15,2)`, producing phantom `ALTER COLUMN TYPE` operations on every plan.

4. Then proceed with the full dev cycle (RED → GREEN → VERIFY).

### Implementation

- Update the introspection query to decode `atttypmod` for `numeric` columns into `numeric(precision,scale)`.
- Also handle `varchar(N)` / `char(N)` if not already covered (`atttypmod - 4` for character types).
- Ensure `schema-flow generate` outputs the full type with precision.
- After a successful run, `schema-flow plan` must report 0 operations for unchanged numeric columns.
