# T-054: Column-level check sugar

- **Status**: TODO
- **Milestone**: 18 — Missing Feature Implementation
- **Depends**: T-006, T-009
- **PRD Reference**: §4.1

## Description

Add `check` string field to ColumnDef (PRD §4.1). When a column has `check: "expr"`, the parser should generate a CheckDef with auto-generated name `chk_<table>_<column>` and the given expression. This is syntactic sugar -- the result is a normal check constraint on the table. Add parser tests and a planner test verifying the generated check appears in the plan.
