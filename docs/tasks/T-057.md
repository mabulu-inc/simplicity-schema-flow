- **Completion Notes**: Added `schema` field to ForeignKeyRef. Parser reads it from YAML, planner uses it for REFERENCES clause (defaults to pgSchema), introspect reads foreign_schema from pg catalog, drift compares FK schema references.

## Description

Add `schema` field to ForeignKeyRef (PRD §4.1). When a foreign key references a table in a different schema, the generated SQL must use `REFERENCES other_schema.table(column)`. If the schema field is omitted, default to the current target pgSchema. Introspection must read the referenced schema for FK constraints. Drift must compare FK schema references.
