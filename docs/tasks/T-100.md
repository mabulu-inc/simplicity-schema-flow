## Description

Running `schema-flow generate --help` currently executes the generate command instead of showing help. Every CLI command should support `--help` as a flag that short-circuits execution and prints command-specific usage.

**Requirements:**

- Every command (`run`, `run pre`, `run migrate`, `run post`, `plan`, `validate`, `status`, `init`, `generate`, `sql`, `erd`, `drift`, `baseline`, `down`, `contract`, `expand-status`, `new`, `lint`) must support `--help`
- `--help` must prevent the command from executing (no DB connection, no file writes)
- Help output must include: command description, all relevant flags with descriptions and defaults, usage examples
- Example: `schema-flow generate --help` should show `--db`, `--dir`, `--output`/`--output-dir`, `--seeds`, `--verbose`, `--json`, etc.
- The top-level `schema-flow --help` (already working) should remain unchanged

## Produces

- `--help` handler in CLI arg parser or per-command dispatch
- Behavioral tests: `--help` on each command prints help text and exits 0 without side effects
