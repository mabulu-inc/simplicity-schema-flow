# T-092: Idempotent RLS policy creation

- **Status**: TODO

- **Complexity**: light

## Description

The planner emits `CREATE POLICY "<name>" ON "<schema>"."<table>" ...` without any idempotency guard. If the policy already exists, the statement fails with `42710` (duplicate_object).

## Current state

- `src/planner/index.ts` line 1414: `CREATE POLICY "${policy.name}" ON "${pgSchema}"."${table}" ...`
- PostgreSQL does not support `CREATE OR REPLACE POLICY` or `CREATE POLICY IF NOT EXISTS`

## Fix

Prefix the `CREATE POLICY` with `DROP POLICY IF EXISTS`:

```sql
DROP POLICY IF EXISTS "<name>" ON "<schema>"."<table>";
CREATE POLICY "<name>" ON "<schema>"."<table>" ...
```

This is safe because the policy is immediately recreated with the desired definition. The planner already uses drop+recreate for policy changes, so this just adds the `IF EXISTS` guard to the drop.
