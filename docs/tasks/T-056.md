# T-056: Seeds enhancements

- **Status**: TODO
- **Milestone**: 18 — Missing Feature Implementation
- **Depends**: T-009, T-010
- **PRD Reference**: §4.1

## Description

Add seed conflict strategy and SQL expression support (PRD §4.1): (1) `seeds_on_conflict` field on TableSchema -- when set to `"DO NOTHING"`, seed operations generate `INSERT ... ON CONFLICT DO NOTHING` instead of upsert. Parser must accept the field. Planner must pass it through to add_seed operations. Executor must vary the generated SQL. (2) `!sql` YAML custom tag support in seed values -- values tagged with `!sql` should be rendered as raw SQL expressions (not string literals) in INSERT statements. Add parser, planner, and executor tests.
