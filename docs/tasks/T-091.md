## Description

The planner emits `CREATE TRIGGER "<name>" ...` without any idempotency guard. If the trigger already exists on the table, the statement fails with `42710` (duplicate_object).

## Current state

- `src/planner/index.ts` lines 1290, 1292: `CREATE TRIGGER "${trigger.name}" ...`
- PostgreSQL 14+ supports `CREATE OR REPLACE TRIGGER`

## Fix

Use `CREATE OR REPLACE TRIGGER` (PG 14+), or for broader version support, prefix with `DROP TRIGGER IF EXISTS` + `CREATE TRIGGER`:

**Option A (PG 14+):**

```sql
CREATE OR REPLACE TRIGGER "<name>" ...
```

**Option B (broader compatibility):**

```sql
DROP TRIGGER IF EXISTS "<name>" ON "<schema>"."<table>";
CREATE TRIGGER "<name>" ...
```

Option A is preferred if the project targets PG 14+ (which is reasonable given PG 14 EOL is Nov 2026). Option B is safer for older versions but has a brief window where the trigger doesn't exist.
