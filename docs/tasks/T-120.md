## Description

Seeds currently execute one SQL statement per row, producing one identical `Seeded: tablename` log line per record. A table with 7 seed rows = 7 round-trips + 7 identical log lines. The output is noisy and uninformative, and per-row execution is unnecessarily slow.

Two changes needed:

1. **Batch execution**: Replace per-row `INSERT ... ON CONFLICT` with bulk operations per table. Pass all seed data as a JSONB recordset parameter. Execute one UPDATE for changed rows (using `IS DISTINCT FROM` to skip unchanged) and one INSERT with `WHERE NOT EXISTS` for new rows — two round-trips per table instead of N.

2. **Summary output**: Replace N identical `Seeded: tablename` lines with a single summary line per table showing what happened: `Seeded: units (3 inserted, 2 updated, 2 unchanged)`.

The planner should emit a single `seed_table` operation per table (not one `add_seed` per row), and the executor should handle the bulk SQL and return row-level counts.

Close GitHub issue #15 after task completion.
