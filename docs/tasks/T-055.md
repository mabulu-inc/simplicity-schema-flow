Add `description` as an alias for `comment` on all schema types that support comments (PRD §4.1): TableSchema, ColumnDef, IndexDef, CheckDef, UniqueConstraintDef, TriggerDef, PolicyDef, EnumSchema, FunctionSchema, ViewSchema, MaterializedViewSchema, RoleSchema. Parser should accept `description` and map it to `comment`; if both are present, `comment` wins. Add parser tests for each schema type.
