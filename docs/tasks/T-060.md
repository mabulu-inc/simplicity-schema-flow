# T-060: E2E: Tables

- **Status**: TODO
- **Milestone**: 20 — E2E Tests: Schema Object Types
- **Depends**: T-059
- **PRD Reference**: §4.1

## Description

Create `test/e2e/tables.test.ts` with behavioral tests for all table features (PRD §4.1). Tests should write YAML, run migration, and verify DB state. Cover: (1) Table creation with all common column types (uuid, text, varchar, integer, bigint, numeric, boolean, timestamptz, jsonb, bytea, text[]), (2) Column defaults (literal, expression like `now()`, `gen_random_uuid()`), (3) Nullable vs non-nullable columns, (4) Single-column primary key, (5) Composite primary key with `primary_key: [col1, col2]`, (6) Custom primary key name (`primary_key_name`), (7) Column-level `unique: true`, (8) Custom unique name (`unique_name`), (9) Column comments, (10) Table comment, (11) Generated columns (`generated: "price * quantity"`), (12) description alias for comment. Each test must verify the resulting DB state by querying pg_catalog/information_schema.

- `test/e2e/tables.test.ts`
