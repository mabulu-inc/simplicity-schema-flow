# T-115: Fix drift false positive for table-level grants without columns field

## Description

`schema-flow drift` reports a false positive immediately after a clean `schema-flow run` when a table grant is declared without a `columns:` field (table-level grant). The grant is applied correctly, but drift reports it as `missing_in_yaml`.

Grants that specify explicit `columns:` drift clean. The issue is that the drift comparison logic does not match a live table-level grant back to a YAML grant declaration that omits the `columns:` field. The matcher likely requires exact structural equality including the columns key, rather than treating an absent `columns:` as "all columns" (i.e., a table-level grant).

After `schema-flow run` on a fresh database with a table-level grant (no `columns:`), `schema-flow drift` must report zero drift.

Close GitHub issue #13 after task completion.

## Produces

- `src/drift/index.ts` — fix grant matching to treat missing `columns:` as table-level grant
- Tests covering: zero drift when table-level grant (no `columns:`) is declared and applied
