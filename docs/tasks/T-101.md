# T-101: Exclude system and extension objects from introspection

- **Status**: DONE
- **Milestone**: 5 — CLI
- **Depends**: none
- **PRD Reference**: §10, §11.1
- **Complexity**: standard
- **Completed**: 2026-03-18
- **Completion Notes**: Added NOT EXISTS pg_depend/pg_extension filters to all introspection queries (tables, enums, functions, views, materialized views). Excluded superusers and known cloud-provider system roles from getExistingRoles. All filtering happens at the SQL query level in src/introspect/index.ts.

## Description

`schema-flow generate` currently outputs YAML files for PostgreSQL extension-owned objects (e.g. functions installed by `uuid-ossp`, `pgcrypto`, `postgis`) and system roles (e.g. `postgres` superuser). These are not user-defined and should not be scaffolded — they create noise and can cause errors if users try to manage them declaratively.

This also impacts **drift detection and diff checks** — extension objects appear as `missing_in_yaml` drift items, producing false positives that obscure real drift.

**Requirements:**

- Filter out extension-owned objects (tables, enums, functions, views) by checking `pg_depend` / `pg_extension` — any object that depends on an extension should be excluded
- Filter out system roles: exclude `postgres` superuser and any role with `rolsuper = true` or matching known system patterns (e.g. roles created by cloud providers like `rds_superuser`, `cloudsqlsuperuser`)
- Filtering must apply in the introspection layer (`src/introspect/index.ts`) so it affects `generate`, `drift`, `diff`, and any other consumer of introspection data
- No CLI flag needed — this should always be the default behavior

## Approach

- For tables, enums, functions, views, materialized views: add a `NOT EXISTS` subquery against `pg_depend d JOIN pg_extension e ON d.refobjid = e.oid WHERE d.objid = <object_oid> AND d.deptype = 'e'`
- For roles: add `AND NOT rolsuper` to the existing `WHERE rolname NOT LIKE 'pg_%'` filter

## Produces

- Updated introspection queries in `src/introspect/index.ts`
- Integration tests confirming extension functions (e.g. from `uuid-ossp`) and the `postgres` role are excluded from generate and drift output
