# T-052: force_rls table attribute

- **Status**: TODO
- **Milestone**: 18 — Missing Feature Implementation

## Description

Add `force_rls` boolean to TableSchema and MixinSchema (PRD §4.1, §4.8). When `force_rls: true`, the planner should emit `ALTER TABLE ... FORCE ROW LEVEL SECURITY` in addition to `ENABLE ROW LEVEL SECURITY`. Introspection must read `relforcerowsecurity` from `pg_class`. Drift must detect when force_rls differs. Parser must accept and validate the field. Update executor to handle the new SQL.
