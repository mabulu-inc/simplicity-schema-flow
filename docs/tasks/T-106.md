## Description

`schema-flow plan` always emits `set_comment` operations for index comments and `create_function` for functions, even when they match the live database. This breaks the idempotency guarantee.

### Ralph must first act as product manager

1. Verify the bug is valid by reading the introspection and planner code.
2. For index comments: confirm whether index comments are being read from `pg_description` joined via `pg_class` for index objects.
3. For functions: confirm whether the planner is comparing function bodies with proper normalization (PostgreSQL may normalize whitespace/formatting).
4. Comment on GitHub issue #3 with findings (valid/invalid, scope assessment, plan of action).
5. Then proceed with the full dev cycle (RED → GREEN → VERIFY).

### Implementation

- Fix index comment introspection to read from `pg_description` for index objects.
- Fix function comparison to handle PostgreSQL's body normalization (trailing newlines, whitespace differences).
- After a successful run, `schema-flow plan` must report 0 operations for unchanged index comments and functions.

- Updated introspection/planner code
- E2E tests: index comments and functions round-trip with 0 phantom operations

## Completion Notes

Fixed two idempotency bugs:

1. **Index comments**: Added `obj_description(i.oid, 'pg_class')` to the index introspection query in `src/introspect/index.ts` to read existing index comments from PostgreSQL catalog. Updated `diffIndexes` in `src/planner/index.ts` to compare desired vs existing comments before emitting `set_comment` operations.

2. **Functions always re-planned**: Replaced the unconditional `create_function` emission in `diffFunctions` with a `functionNeedsUpdate()` comparison that checks body (with whitespace normalization), language, returns, security, volatility, parallel, strict, leakproof, cost, rows, set, and args. Only emits `create_function` when an actual difference exists. Also skips `set_comment` and `grant_function` when unchanged.

11 tests added (8 unit tests for planner logic + 3 E2E tests with real PostgreSQL).
