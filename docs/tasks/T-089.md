# T-089: Idempotent table creation

- **Status**: TODO
- **Milestone**: 28 — Idempotent Pipeline Operations
- **Depends**: (none)
- **PRD Reference**: §6.1 (pipeline step: Tables)
- **Complexity**: light

## Description

The planner emits `CREATE TABLE "<schema>"."<table>" (...)` without `IF NOT EXISTS`. While the diff engine normally prevents re-creation (it only emits CREATE when introspection shows the table is missing), this is not a guarantee — introspection mismatches, partial runs, or external table creation can cause `42P07` (duplicate_table) errors.

## Current state

- `src/planner/index.ts` line 594: `CREATE TABLE "${pgSchema}"."${table.table}" (...)`

## Fix

Add `IF NOT EXISTS` to the `CREATE TABLE` statement:

```sql
CREATE TABLE IF NOT EXISTS "<schema>"."<table>" (...)
```

This is safe because:

- If the table doesn't exist, it creates it normally
- If the table already exists, it's a no-op (the subsequent diff/alter logic handles convergence)

## Produces

- Updated table creation SQL in the planner to include `IF NOT EXISTS`
- Test verifying that running the pipeline twice does not error on table creation
