# T-111: Support NULLS NOT DISTINCT on unique constraints (PostgreSQL 15+)

- **Status**: TODO
- **Milestone**: 34 — Feature: NULLS NOT DISTINCT
- **Depends**: none
- **PRD Reference**: §4
- **GitHub Issue**: #8
- **Complexity**: standard

## Description

Add support for the `nulls_not_distinct` option on unique constraints, introduced in PostgreSQL 15. This allows nullable columns in a unique constraint to treat NULL values as equal (i.e., only one NULL is permitted per unique group).

Currently the parser, planner, executor, and introspector have no awareness of this modifier. Users must work around it with raw SQL post-migration scripts, breaking the declarative model.

## Requirements

1. **Parser** — Accept `nulls_not_distinct: true` in unique constraint YAML definitions. Update the `UniqueConstraintDef` type to include the optional boolean field.

2. **Planner / SQL generation** — Emit `NULLS NOT DISTINCT` in both:
   - `CREATE TABLE` inline constraints: `CONSTRAINT "name" UNIQUE NULLS NOT DISTINCT (cols)`
   - `CREATE UNIQUE INDEX CONCURRENTLY` for existing tables: `... ON table (cols) NULLS NOT DISTINCT`

3. **Introspection** — Read `indnullsnotdistinct` from `pg_index` (PG 15+) and populate the parsed model so the planner can diff against it. For PG < 15 the column does not exist; introspection should default to `false`.

4. **Diff / convergence** — Detect when the desired `nulls_not_distinct` value differs from the introspected value and plan an appropriate alter (drop + recreate). This covers both `diff` (plan display) and `run` (plan execution).

5. **Scaffold / generate** — The scaffold (`src/scaffold/index.ts`) passes introspected constraint objects through to YAML. Once introspection populates `nulls_not_distinct`, generate will include it automatically. Verify with a round-trip test: create a `NULLS NOT DISTINCT` constraint in the DB, run generate, and confirm the output YAML contains `nulls_not_distinct: true`.

6. **Documentation** — Update all relevant docs to reflect the new option:
   - `README.md` — add `nulls_not_distinct` to the unique constraints example if unique constraints are shown
   - `docs-site/src/content/docs/schema/tables.md` — add `nulls_not_distinct: true` to the unique constraints YAML example and describe the option
   - Any other docs-site pages that reference unique constraint syntax (e.g., mixins, zero-downtime, rollback, destructive-protection, execution-phases)

7. **Close GitHub issue #8** after task completion.
