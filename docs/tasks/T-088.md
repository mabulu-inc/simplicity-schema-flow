# T-088: Idempotent enum creation

- **Status**: TODO
- **Milestone**: 28 — Idempotent Pipeline Operations
- **Depends**: —
- **PRD Reference**: §6.1 (pipeline step: Enums)

## Description

The planner emits `CREATE TYPE "<name>" AS ENUM (...)` without any idempotency guard. If the enum already exists in the database (e.g., from a previous partial run, manual creation, or introspection mismatch), the statement fails with error `42710` (duplicate_object).

## Current state

- `src/planner/index.ts` line 257: `CREATE TYPE "${desiredEnum.name}" AS ENUM (${values})`
- PostgreSQL does not support `CREATE TYPE IF NOT EXISTS`

## Fix

Wrap the `CREATE TYPE` in a `DO` block that checks `pg_type`:

```sql
DO $$ BEGIN
  IF NOT EXISTS (SELECT 1 FROM pg_type WHERE typname = '<name>') THEN
    CREATE TYPE "<name>" AS ENUM (...);
  END IF;
END $$
```

## Produces

- Updated enum creation SQL in the planner to use idempotent pattern
- Test verifying that running the pipeline twice does not error on enum creation
