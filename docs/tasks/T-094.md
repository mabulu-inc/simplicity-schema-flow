## Description

The planner emits `ALTER TABLE ... ADD CONSTRAINT "<name>" FOREIGN KEY (...)` and `ALTER TABLE ... ADD CONSTRAINT "<name>" CHECK (...)` without idempotency guards. If the constraint already exists, the statement fails with `42710` (duplicate_object).

## Current state

- `src/planner/index.ts` line 637: FK on initial table creation
- `src/planner/index.ts` line 837: FK added via diff
- `src/planner/index.ts` line 979: NOT NULL check constraint (NOT VALID)
- `src/planner/index.ts` line 1122: Check constraints
- PostgreSQL does not support `ADD CONSTRAINT IF NOT EXISTS`

## Fix

Wrap each `ADD CONSTRAINT` in a `DO` block that checks `pg_constraint`:

```sql
DO $$ BEGIN
  IF NOT EXISTS (SELECT 1 FROM pg_constraint WHERE conname = '<name>') THEN
    ALTER TABLE "<schema>"."<table>" ADD CONSTRAINT "<name>" FOREIGN KEY (...);
  END IF;
END $$
```

Same pattern for check constraints:

```sql
DO $$ BEGIN
  IF NOT EXISTS (SELECT 1 FROM pg_constraint WHERE conname = '<name>') THEN
    ALTER TABLE "<schema>"."<table>" ADD CONSTRAINT "<name>" CHECK (...);
  END IF;
END $$
```

For `NOT VALID` constraints, preserve the `NOT VALID` flag inside the guard.
