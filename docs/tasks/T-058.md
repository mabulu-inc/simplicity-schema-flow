# T-058: Expand reverse and batch_size

- **Status**: TODO
- **Milestone**: 18 — Missing Feature Implementation
- **Depends**: T-015
- **PRD Reference**: §4.1, §11.3

## Description

Add `reverse` and `batch_size` fields to ExpandDef (PRD §4.1, §11.3). `reverse` is a SQL expression for the dual-write trigger's new->old direction (enables backward-compatible writes during migration). `batch_size` overrides the default 1000 rows per backfill batch. Parser must accept both fields. Expand planner must use `reverse` in trigger function generation and `batch_size` in backfill operations.
