## Description

The default output of `schema-flow run` only shows a summary line with total operation counts. There is no indication of what actually changed. Users must use `--verbose` to see details, which dumps every operation including hundreds of no-ops.

Implement three output verbosity modes:

1. **`--quiet`** — summary line only (or nothing on zero changes)
2. **Default (no flags)** — show only operations that made a change, plus a summary line (e.g., "Migration complete: 4 changes (761 unchanged)")
3. **`--verbose`** — all operations including no-ops/skips (current behavior)

Each change line should describe what happened: "Created table: X", "Added index: Y", "Applied policy: Z", "Granted ... on X to Y", etc.

This applies to `schema-flow run` and `schema-flow drift --apply`. The `schema-flow plan` command already shows planned operations and does not need changes.
